# 06_ADR — 架构决策记录

## 职责

记录**重要技术决策**及其理由，避免未来人类与 AI Agent 默默推翻。

## 何时写 ADR

- 选择或更换主要依赖 / 平台
- 变更系统边界或数据所有权
- 影响安全、隐私或合规的决策
- 难以回退或再议成本高的事项

## 规则

- ADR 精神上是追加式：用新 ADR 废止旧决策，勿改写历史。
- 链接相关 Architecture 与 Spec。
- 状态取值：Proposed（提议）| Accepted（已接受）| Deprecated（废弃）| Superseded（被替代）
