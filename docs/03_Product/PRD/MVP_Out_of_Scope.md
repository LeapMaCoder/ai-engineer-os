# MVP Out of Scope — 强制禁止范围

> MVP **只**服务 Primary Problem（匹配路径 + 持续可信反馈 → 不知下一步 / 难坚持）。  
> Hard No 已经 Founder 确认保留。

## 1. 明确禁止（Hard No）— Founder 确认

| 禁止域 | 具体禁止 |
|--------|----------|
| **课程平台** | 课程货架、章节商城、大量 SKU、以「上新课」当增长 |
| **社区** | 论坛、动态圈、同学匹配、UGC 优先 |
| **招聘** | 内推、岗位匹配、简历、保证就业/涨薪 |
| **IDE** | 在线 IDE、工程管理工作台、调试器产品化 |
| **代码生成工具** | 以代写/生成项目代码为主价值 |
| **复杂游戏系统** | 联赛、段位赛、重度排行、强诱导连胜经济 |

另禁止：企业培训 LMS、残缺免费逼付费（原则 9）。

## 2. 本阶段禁止的工作类型

- 编写业务代码  
- 技术架构 / 技术选型 ADR（过早）  
- 数据库设计  
- UI / 视觉 / 高保真线框  

## 3. Not Now（可研究，不进本 MVP 交付）

- 深度知识图谱产品化  
- 完整测评认证体系  
- 多语言内容版图  
- 付费套餐与定价实验（先有 Monetization / WTP Signal）  

## 4. 决策句

> 任何提案若更像「课平台 / 社区 / 招聘 / IDE / 代码生成 / 复杂游戏」，默认拒绝，除非 Founder 改写 Primary Problem 并更新 [[Decision_Log]]。
