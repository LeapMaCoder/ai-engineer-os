# MVP Vision — 第一个 MVP 要证明什么

> 证据标注：Confirmed / Hypothesis / Unknown  
> **不假设用户一定付费。** 付费是成长闭环成立后的自然结果假设。

## 1. MVP 一句话目标

证明：LeapMa 能让目标用户在**目标驱动**下完成首次有效成长体验（评估 → 路径 → 行动 → 反馈 → 可见），并愿意重复进入循环直至下一目标——免费层本身形成可留存闭环，付费是「更强成长」而非「解开残缺体验」。

级别：**Hypothesis**（持续验证；**不**阻塞 MVP PRD 推进）

## 2. 要证明什么价值

| 价值主张 | 用户感知 | 级别 |
|----------|----------|------|
| 成长反馈 | 「我的问题被看见了，错误被说清楚了」 | **Hypothesis** |
| 个性化下一步 | 「我不用自己猜接下来练什么」 | **Hypothesis** |
| 长期坚持可启动 | 「我愿意明天再来一次短会话」 | **Hypothesis** |
| 情境建模 | 「先在故事/困境里懂了概念，再做验收题」 | **Hypothesis**（D-056 / US-05） |
| 能力开始可见 | 「我比昨天更清楚自己会/不会什么」 | **Hypothesis** |

**不是要证明：** 课程最多、题库最大、社区最热闹。  
级别：与愿景对齐 — **Confirmed**（战略选择已文档化）；市场是否买账 — **Unknown**

## 3. 要验证的核心假设

| ID | 假设 | 级别 | MVP 如何碰 |
|----|------|------|------------|
| H2 | 坚持失败主因含反馈缺失 + 日程 | Unvalidated | 看断裂是否因「无反馈/不知下一步」减少 |
| H3 | 能力可见降低焦虑乱跳 | Unvalidated | 轻量「会/不会」可见是否提升回访 |
| H4 | 更愿为反馈/效率付费 | Unvalidated | 观察付费意愿信号，不做假想承诺 |
| H5 | AI 反馈可用但幻觉毁信任 | Unvalidated | 信任/无帮助反馈护栏 |
| H6 | 动态下一步 > 纯固定课表 | Unvalidated | MVP 至少提供「按卡住调整下一步」 |
| H8 | 职场补技能更优首发 ICP | Unvalidated | MVP 文案与场景优先该群，但可对照 |
| M1 | 免费完整闭环可留存 | **Hypothesis** | Free 必须跑通目标驱动整环 |
| M2 | 付费来自「天花板」而非「残缺」 | **Hypothesis** | 见 Freemium；早期看 Monetization Signal |
## 4. 用户第一次体验应获得什么（Activation）

第一次有效体验（Aha）方向（目标驱动，非 UI）：

1. 设定或澄清一个学习目标  
2. 相对目标有一次轻量能力位置感知（评估）  
3. 得到个性化「下一步」并完成一次学习行动  
4. 收到可理解的 AI 反馈  
5. 看到相对目标的轻量成长可见信号  

级别：**Hypothesis**

## 5. 免费用户价值

免费用户应获得：

- **可运行的目标驱动成长闭环**（目标 → 评估 → 路径 → 行动 → 反馈 → 可见 → 下一目标）  
- 足够启动习惯的反馈与下一步  
- 诚实、有边界的 AI 辅导（基础深度）

免费用户**不是**被限制用户，而是未来价值用户。  
见原则 9 Growth Before Monetization：[[Product_Principles]]  
级别：**Hypothesis**（商业可持续性 — **Unknown**）

## 6. 付费用户价值

付费用户应获得「同一闭环上的更强成长」，例如方向上：

- 更深 / 更稳的个性化路径与诊断  
- 更强的卡住恢复（更细反馈、更多上下文陪伴）  
- 更清晰的中长期能力可见与目标对齐  
- 更高频率或更高强度的导师辅导额度（价值差，而非拆断闭环）

级别：具体打包方式 **Hypothesis**；价格带 **Unknown**；转化率 **Unknown**

## 7. 三个灵魂问题（本文件回答）

| 问题 | MVP 方向性回答 | 级别 |
|------|----------------|------|
| 为什么留下？ | 目标清晰 + 反馈 + 下一步，而非又一份课单 | **Hypothesis** |
| 为什么持续使用？ | 目标未完成 + 路径开放环 + 进展可见 | **Hypothesis** |
| 为什么未来愿意付费？ | 免费已证明有用；升级突破天花板（先看 Monetization Signal） | **Hypothesis**（非保证） |

## 8. 非承诺

- 不承诺 MVP 即盈利  
- 不承诺全员付费  
- 不承诺跳过持续验证即可「永久锁定」ICP（可修订）  
- 访谈是 Continuous Validation，**不**阻塞 MVP PRD Definition  

## 9. Founder Review 检查点

- [ ] 是否同意「免费必须跑通闭环」？  
- [ ] 是否同意付费差在「强度/深度」而非「残缺解锁」？  
- [ ] 首发场景是否暂定职场补技能（H8）？
