# Non-Goals — MVP 明确不做

列出不做，是为了保护闭环验证。  
若 Founder 要坚持做某项，须写明牺牲了哪条原则 / 哪个假设验证。

## 1. 不做清单

| 不做 | 原因 | 级别 |
|------|------|------|
| 完整课程超市 / 大量 SKU | 优化库存会偏离成长价值 | 对齐愿景 — **Confirmed**（战略）；用户是否仍要课 — **Unknown** |
| 社区 / 论坛优先 | 不替代反馈闭环；运营成本高 | **Hypothesis** |
| 复杂排行榜 / 联赛 | 空转与羞耻驱动风险；H7 | **Hypothesis** |
| 企业培训 / LMS | 原则 7 垂直优先个人成长 | **Confirmed**（原则） |
| 多语言全面内容覆盖 | MVP 验证闭环，不做内容版图 | **Hypothesis** |
| 为转化而残缺免费 | 违反原则 9 Growth Before Monetization | **Confirmed**（Founder 方向） |
| 技术/UI/API/库表设计 | 本阶段禁止 | **Confirmed**（阶段约束） |
| 保证付费成功 | 禁止假设一定付费 | **Confirmed**（研究纪律） |

## 2. 可以说「以后可能」但不进 MVP

- 深度知识图谱  
- 团队/班级  
- 认证考试  
- 招聘联动  

级别：均为 **Unknown** 机会，不作承诺。

## 3. 决策句

> MVP 只回答：用户会不会因为「反馈 + 下一步 + 可见进展」而回来；以及免费有用后会不会为「更强成长」付费。
