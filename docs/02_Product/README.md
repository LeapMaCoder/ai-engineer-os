# 02_Product

## Purpose

Define **what** we build for **whom**, and what success looks like.

Product sits after Vision and Research, before Specifications:

Vision → **Product** → Specification → Architecture → Code → Test

## Contains

- PRDs
- Personas and jobs-to-be-done
- User journeys / flows (problem-level)
- Success metrics and acceptance themes
- Explicit non-goals

## Rules

- Every PRD must link to Vision (and relevant Research).
- Product docs describe outcomes and constraints, not APIs or schemas.
- No implementation without a Product doc that justifies the work.
