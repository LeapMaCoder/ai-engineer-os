# 02_Product

## 职责

定义为**谁**、做**什么**，以及成功长什么样。

Product 位于 Vision 与 Research 之后、Specification 之前：

Vision → **Product** → Specification → Architecture → Code → Test

## 包含

- PRD
- ICP 决策
- 用户画像与要完成的工作（JTBD）
- 用户旅程 / 流程（问题层）
- 成功指标与验收主题
- 明确的非目标

## 规则

- 每份 PRD 必须链接 Vision（以及相关 Research）。
- 产品文档描述结果与约束，不写 API 或库表。
- 没有能证明价值的产品文档，禁止实现。
