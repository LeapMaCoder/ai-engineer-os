## Growth Loop Mapping（强制）

单 Spec 覆盖 **GL-1…GL-8 最小路径**。每环只做「最小必要」，不追求完整体系（D-039）。

| GL | 本 Spec 是否服务 | 最小必要做什么 |
|----|------------------|----------------|
| GL-1 目标设定 | ✅ | 澄清用户大致学习方向 / 何为成功的一句话目标意图（不必深度工作坊） |
| GL-2 能力评估 | ✅ | **轻量位置感以对话推断为主**；探针**默认跳过**，仅目标极度模糊时建议 1 题且始终可跳过；不做完整测评（OQ-001 / BR-011…013） |
| GL-3 个性化路径 | ✅ **主** | 给出与目标相关的**下一步学习行动**（可陈述、可执行） |
| GL-4 学习行动 | ✅ | **默认短练习为主，目标向提问为辅**；一次可在单次会话内结束（OQ-002 Confirmed） |
| GL-5 AI 反馈 | ✅ **主** | 可理解的对错或改进点；不确定时坦诚边界（见 AI Behavior；不可削弱） |
| GL-6 能力提升 | ✅ | 用户能感到本轮有**真实微小进展**（相对目标） |
| GL-7 成长可见 | ✅ | **用户能指出 ≥1 点相对目标的具体进展**；系统可协助对照目标陈述；禁止假等级/空夸奖代替（OQ-003 Confirmed） |
| GL-8 下一目标 | ✅ | 给出为何再来：下一小步或下一目标意图（续环，非独立目标管理系统） |

> 主战场对齐 Primary：**GL-3 + GL-5**；GL-1/2 为定向最低必要；GL-6…8 承接坚持与续环。

## Scope

### In Scope

1. 首次成长体验的端到端**价值行为**：定向 → 行动+反馈 → 进展感知与续环意图  
2. 免费用户跑通上述最小路径（US-04 / AC-04）  
3. AI 在 GL-5 的允许/禁止/不确定边界（诚实反馈）  
4. 与 AC-01…AC-04 对齐的可测试验收（可细化，不可削弱）

### Out of Scope（继承 Hard No + Not Now）

**Hard No（默认排除）：**

- 课程平台 / 课程货架 / 大量 SKU  
- 社区 / 论坛 / UGC 优先  
- 招聘 / 内推 / 简历 / 保证就业  
- 在线 IDE / 工程管理工作台  
- 以代写/生成项目代码为主价值  
- 复杂游戏系统（联赛、重度排行、强诱导连胜经济）  
- 企业培训 LMS；残缺免费逼付费  

**本 Spec 另排除（D-039 / Later）：**

- 完整能力评估体系、深度目标工作坊、复杂个性化引擎  
- 独立「下一目标管理系统」产品化  
- 付费套餐 / 定价实验（可观察 Monetization Signal，不要求付费）  
- 本 Spec **不**定义技术栈/UI；实现前须另写最小 Architecture / ADR（不在本文件展开）

## User Flow（禁止 UI）

价值步骤（非页面/控件）— **方案 1：教练对话，非问卷墙**：

1. 用户进入；用**一句话**表达想提升什么（单输入 / 对话式，非多题表单）  
2. 系统以**对话推断**给出轻量位置感，并**尽快**给出可陈述的下一步；探针**默认跳过**——仅目标极度模糊时建议 1 题，且用户始终可跳过  
3. 用户获得**可陈述的下一步**学习行动（与目标相关）  
4. 用户完成一次单次会话内可结束的学习行动——**主路径为短练习**；目标向提问为辅  
5. 用户获得可理解的 AI 反馈；若不确定则收到坦诚边界与有效下一步引导  
6. 用户回顾本轮，能指出相对目标的 **≥1 点具体进展**（系统可协助对照目标陈述；禁止假等级/空夸奖）  
7. 用户清楚为何再来（下一小步或下一目标意图），可续入下一轮定向/行动  

```mermaid
flowchart LR
  G1[GL-1 一句话目标] --> G2[GL-2 对话推断\n探针默认跳过]
  G2 --> G3[GL-3 下一步]
  G3 --> G4[GL-4 短练习为主]
  G4 --> G5[GL-5 可信反馈]
  G5 --> G6[GL-6 微小进展]
  G6 --> G7[GL-7 用户指出进展]
  G7 --> G8[GL-8 为何再来]
  G8 -.-> G1
```

**推荐价值闭环（对齐 PRD）：**  
一句话目标 →（极少可选补充）→ 下一步 → 短练习 + 可信反馈 → 进展可见并续环；全程免费可跑通。

## AI Behavior（GL-5 可信反馈）

对齐产品原则「诚实 AI」与 AC-02。**OQ 定稿不得削弱本表。**

| ID | 允许 | 禁止 | 不确定时 |
|----|------|------|----------|
| AI-001 | 指出对错、局部错误原因、可执行的改进点；对齐用户当前下一步 | 伪造权威、编造未验证事实冒充确定答案 | 明确说明不确定点，给出可验证的下一步（如缩小问题、换角度练、需要用户补充的信息） |
| AI-002 | 在目标相关范围内讲解概念、纠错、引导改错再练 | 以代写完整项目 / 大量生成代码**作为主价值**（Hard No） | 说明能力边界；引导用户自行完成关键练习步骤 |
| AI-003 | 反馈语言可理解、可行动，服务「知道如何改」 | 空洞鼓励、无关说教、诱导付费才能获得基本反馈 | 坦诚「此处无法可靠判断」，仍给出免费可用的续练方向 |
| AI-004 | 帮助用户把本轮反馈与目标进展关联（支撑 GL-6/7） | 用虚假「等级暴涨」或游戏化诱导代替真实进展 | 若进展证据不足，如实说明，并建议可产生证据的下一行动 |

## Acceptance Criteria

对齐 PRD AC-01…AC-04；可细化通过信号，**不可削弱**价值闭环。OQ 定稿只澄清形态，不改变下列准则强度。

| AC ID | 准则（价值闭环） | 通过信号（可测试意图） |
|-------|------------------|------------------------|
| AC-01 | Given 用户带着模糊或大致学习方向进入，When 完成最小定向（一句话目标 + 对话推断位置感；**不要求先完成问卷**），Then 用户能主动说出下一步学习行动是什么，且该步与其目标相关。 | 用户口述/复述「我下一步要做 X」；X 与目标相关；路径符合 BR-009/011/012/013 |
| AC-02 | Given 用户已有明确下一步，When 完成一次可在单次会话内结束的学习行动，Then 用户获得可理解的反馈（对错或改进点）；若系统不确定，用户得到坦诚边界而非伪造权威。 | 行动可结束（主路径短练习）；反馈含可理解改进点 **或** 明确不确定声明 + 有效下一步；禁止「假确定」 |
| AC-03 | Given 用户完成至少一轮「下一步→行动→反馈」，When 回顾本轮，Then 用户能指出相对目标的至少一点具体进展，并能说明下一次回来要继续的方向（下一小步或下一目标）。 | 用户能指出 ≥1 点具体进展 + 能说明再来方向；系统协助对照不算削弱用户「能指出」要求 |
| AC-04 | Given 用户未付费，When 走完 US-01～US-03 的最小路径，Then 主价值闭环全部可完成；不出现因未付费而无法获得下一步或可信反馈或进展感知。 | 未付费路径走通 AC-01…03；无「付费墙阻断成长」 |

### 细化（不削弱）

| 细化 ID | 说明 |
|---------|------|
| AC-01a | 「下一步」须是用户可立即开始的学习行动，而非仅抽象课程目录 |
| AC-02a | 反馈须服务改错/再练；纯情绪夸奖不足以通过 |
| AC-03a | 「进展」须相对目标可感知；纯会话时长不算进展 |
| AC-04a | 最小路径上不得插入付费才能解锁的必需步骤 |

## Edge Cases

| ID | 场景 | 期望行为 |
|----|------|----------|
| EC-01 | 用户目标极度模糊（「我想学编程」） | 可**建议** 1 道可跳过补充；仍尽快给出可陈述下一步；不假装完整测评 |
| EC-02 | 用户拒绝澄清、只想「随便练」 | 给出与可见意图最相关的下一步；标注假设；**探针可跳过**（默认） |
| EC-03 | AI 对题目/答案不确定 | AI-001：坦诚边界 + 有效下一步；禁止伪造权威 |
| EC-04 | 用户要求「直接生成整个项目代码」 | 拒绝作为主价值；引导回目标相关短练习 + 反馈（Hard No） |
| EC-05 | 用户一轮后感觉「没进步」 | 诚实承认证据不足；帮助指出可观察的微小证据或建议能产生证据的下一行动（非假进度） |
| EC-06 | 未付费用户触及增强能力 | 免费最小环仍可完成；增强可说明但不阻断 AC-01…03 |
| EC-07 | 用户切换目标中途 | 允许重做最小定向；旧「下一步」可失效并重新给出与新目标相关的下一步 |
| EC-08 | 用户目标已足够具体 | **不展示**探针页；直接 NextStep → 短练习 |
